ここに Claude への指示を書く。

$ARGUMENTS は実行時の引数に置換される。

例: /frontend:module billing
→ $ARGUMENTS が "billing" に置換

```

---

### 各コマンドの役割

#### /frontend:module
```

役割: Feature Sliced Design のモジュール構造を一発生成
用途: 新機能追加時に api/, contract/, model/, ui/ を作成

```

#### /frontend:page
```

役割: App Router のページを作成
用途: 新しいルートを追加

```

#### /frontend:bff
```

役割: BFF ルートハンドラを作成
用途: バックエンド API へのプロキシを追加

```

#### /backend:usecase
```

役割: クリーンアーキテクチャのユースケース構造を生成
用途: 新しいビジネスロジックを追加

```

#### /backend:router
```

役割: FastAPI ルーターを作成
用途: 新しいエンドポイントを追加

```

#### /backend:migration
```

役割: Alembic マイグレーションを作成
用途: DB スキーマ変更時

```

#### /shared:pr
```

役割: PR 本文を自動生成
用途: コミット後の PR 作成

```

#### /shared:api-sync
```

役割: OpenAPI 仕様とコードの整合性チェック
用途: API 変更後の確認

